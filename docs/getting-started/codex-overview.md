## 为什么读这篇

你可能在某个技术群里看到有人提 Codex，点开 OpenAI 官网发现既有网页版又有命令行工具，还有什么 App、IDE 扩展……一堆入口让人看不懂到底用哪个。

这篇文章只做一件事：帮你在 3 分钟内搞清楚 Codex 这个名字背后藏了几个东西、它能帮你做什么、你应该从哪个入口开始。

## 一句话结论

Codex 是 OpenAI 2025 年推出的 AI 编程代理（agent）产品家族，有五个入口，面向个人开发者的起点推荐 Codex CLI。

## Codex 到底是什么，有哪几个入口

"Codex"这个名字 2025 年回归时，已经不再只是一个模型，而是一整个产品家族的总称。你可以把它理解成一套"AI 编程助手的不同皮肤"——核心能力相同，但交互方式各有侧重。

下面是五个入口的全貌：

| 产品 | 形态 | 适合场景 |
|------|------|---------|
| Codex CLI | 终端命令行工具，开源 | 本地开发，日常提交、改 bug、写脚本 |
| Codex App | 桌面客户端（macOS / Windows） | 不想开终端、图形化交互更顺手 |
| Codex Web | chatgpt.com/codex 上的云端版本 | 浏览器里随手用，无需本地安装 |
| Codex IDE 扩展 | VS Code、Cursor、Windsurf 等编辑器插件 | 在编辑器里内嵌使用，不切换窗口 |
| Codex iOS | iPhone / iPad 端 App | 移动端随时查阅、简单任务 |

我第一次看到这个表格时有点懵——为什么要搞这么多入口？后来用了一段时间才明白：这几个入口面向的场景和用户习惯是不一样的，并不是让你全部都用。

## 它能做什么，不能做什么

### 能做的事

Codex 的核心定位是**代理（agent）**，而不是聊天机器人。区别在哪？

打个比方：如果 ChatGPT 是个顾问——你问它"这段代码怎么改"，它给你一段建议，你自己去改——那 Codex 更像是一个跟着你坐在旁边的实习生，你说"帮我把这个函数重构一下"，它直接动手改文件、跑测试、告诉你结果。

具体来说，Codex 能：

- 读取你的代码库，理解项目结构
- 直接修改文件（加功能、改 bug、重构）
- 在你的本地环境执行命令（跑测试、构建、脚本）
- 按你的指令完成多步骤任务，中间不需要你盯着每一步

### 不能做的事

直说吧，Codex 目前有几个明显限制：

- **不能替你做架构决策**：它执行你的想法，但"这个功能该怎么设计"还得你拍板
- **不擅长超大规模跨文件改动**：几个文件之间联动的大重构，容易顾此失彼
- **没有持久记忆**：每次对话是独立的，除非你用 `AGENTS.md` 把项目规范告诉它（这个后面专文讲）
- **需要你审查结果**：它不会犯低级错误，但也不是全知全能的，改完最好看一眼 diff

::: tip 新手常见误区
Codex 不是"AI 替我写完整个项目"的工具。把它当作一个执行力强、不嫌烦的协作者，你来定方向，它来动手——这样用才顺。
:::

## 常见坑

**坑 1：把 Codex CLI 和初代 Codex 混了**

2021-2023 年 OpenAI 有过一个叫 Codex 的代码模型（当时主要是 API），2025 年回归的 Codex 是完全不同的产品，别把旧资料当参考。搜资料时加上"2025"能过滤掉大部分旧内容。

**坑 2：五个入口选错了**

Codex Web 适合临时体验，但功能受限，读写你本地文件的能力也有限。如果你想在真实项目上用，从 Codex CLI 或 Codex App 入手，才能发挥完整能力。
