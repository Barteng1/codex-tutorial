# 风格统一指南

本指南规定网站所有内容的写作、代码、视觉风格。所有 Agent 在产出前必须读完，提交前用末尾的自检清单逐条打勾。

视觉细节（具体颜色、字体粗细、间距像素值）属于 MVP 之后再调优的范围，本指南只规定**结构性**和**写作性**的约束。

## 一、写作调性

整体感觉: **像 B 站上一个稍微有点经验、愿意把自己踩过的坑全告诉你的 up 主**。不是大学教授讲座，也不是营销软文。真诚、好玩、能把人留下来看完。

具体要求：

- **务实优先**: 每个概念都先回答"这有什么用 / 什么时候用 / 不用会怎样"。展示概念的全貌不是目标，让读者用得起来才是。
- **直面困难，可以吐槽**: 遇到坑、限制、奇怪的设计就直说，可以带点情绪。"为啥 Codex 拖文件要按住 Shift？我也没搞懂"比"请按住 Shift 键拖入文件"更让人记住。
- **可以用第一人称分享经历**: "我第一次配 Codex 卡在网络问题上半小时"、"如果你也碰上这种情况，别慌"——这种叙述比纯技术说明更有粘性，鼓励使用。
- **类比可以接地气**: 把 AGENTS.md 比作"塞给新同事的项目说明书"、把审批模式比作"开车要不要每次系安全带都问你"。类比之后回到原本的术语即可。
- **允许节制使用网络化表达**: "一行命令搞定"、"踩过的坑"、"真香"、"翻车"等 B 站常见说法可用，但**每篇限 1-2 处**，超过会变油腻。
- **依然不端架子**: 不写"众所周知"、"显而易见"、"业内人士"。读者就是来学的，假定他什么都不知道。
- **依然不堆 SEO 自媒体黑话**: 不写"小白看过来"、"一文吃透"、"宝藏工具"、"YYDS"、"绝绝子"、"震惊"、"99% 的人不知道"。这些不是 up 主风，是低质量内容工厂的标记。
- **不要每段都写成视频开头**: 不是每段都要"今天来给大家讲讲……"。该开门见山就开门见山。

## 二、人称与语气

- 第二人称统一用"你"，**不用"您"**、不用"读者"、不用"我们"（除非真是作者带读者一起做某件事的语境）。
- 作者自指用"我"，可以分享个人经历，但不用"笔者"、"小编"、"博主"、"up 主"自指。
- 句子优先短，能 15 字说清楚不用 25 字。
- 避免翻译腔："它是一个能够……的工具" → "它能……"
- 动词驱动，少用名词化："对配置进行修改" → "改配置"
- **允许在段落末尾偶尔加一句吐槽或感叹做节奏调剂**，但每段最多一处，不要变成段段都有。

## 三、文章结构模板

每篇文章必须按这个六段式：

```
1. 为什么读这篇    （1-3 句话点出读者的痛点 / 收益）
2. 一句话结论      （30 字以内，让读者带着方向看下文）
3. 详细步骤        （按操作顺序，可分小节）
4. 完整可复现的例子 （必须有，不能省）
5. 常见坑          （至少列 2 个，附解决办法）
6. 下一步          （链接到 1-3 篇相关文章，或推荐进一步动作）
```

例外: 概念解释类文章（如《Codex 是什么》），第 3、4 段可合并为"它能做什么 / 不能做什么"。

## 四、标题规范

- 文章主标题（H1）由 frontmatter 的 `title` 自动生成，正文从 H2 开始。
- 最多用到 H3。超过 H3 说明结构需要拆分。
- 标题不带 emoji、不带句号、不带"！"。
- 标题用陈述句或名词短语；问句仅在少数提点场景使用，全篇不超过 1 处。

## 五、代码块规范

- **必须标语言**: 用 ```` ```bash ````、```` ```typescript ````、```` ```json ````、```` ```toml ```` 等。不带语言标签的代码块视为不合规。
- **命令行示例去掉前导 `$`**: 让读者直接复制。需要区分命令和输出时用注释说明。
- **关键行加注释**: 配置文件、命令、代码示例都要把读者最可能困惑的行用注释解释。
- **超长代码先讲再放**: 超过 20 行的代码块，前面必须有一段说明这段代码做什么、读者重点看哪几行。

正例：

````markdown
```bash
# 全局安装 Codex CLI
npm install -g @openai/codex

# 检查版本（出现版本号即安装成功）
codex --version
```
````

反例：

````markdown
```
$ npm install -g @openai/codex
$ codex --version
```
````
（无语言标注、有 `$` 前缀、缺注释，三个问题）

## 六、术语统一

详细见 `glossary.md`。重点提示：

- "Codex" 单独出现时，默认指 OpenAI 2025 年回归后的整个产品家族。
- 具体到产品时必须写清楚: "Codex CLI"、"Codex App"、"Codex Web"、"Codex IDE 扩展"。
- "agent" 在面向读者的内容中翻译为"代理"或"代码代理"；在 `.agents/` 内部文档中保留"Agent"。
- 全文首次出现的英文技术术语，加中文解释或括号释义；后续可直接使用英文。

## 七、Frontmatter 规范

每篇 Markdown 文章必须有以下字段：

```yaml
---
title: 文章标题（不超过 24 字）
description: 一句话摘要，60-100 字，会用作 SEO 描述
order: 1                    # 在所属章节内的排序，整数
difficulty: 入门             # 入门 | 进阶 | 高级 | 参考
updated: 2026-05-02         # ISO 日期，每次实质修改都要更新
---
```

## 八、文件与目录命名

- 全部使用 kebab-case，如 `install-codex-cli.md`、`first-bug-fix.md`。
- 不允许中文文件名、空格、下划线。
- 章节目录用单数英文名: `getting-started/`、`beginner/`、`intermediate/`、`advanced/`、`reference/`。

## 九、图片规范

- 路径: `public/images/<章节>/<文件名>.png`，文件名 kebab-case。
- 引用: Markdown 语法 `![描述](/images/.../foo.png)`，alt 文本必填且具备描述性。
- 截图统一: 浅色主题、隐藏个人信息、关键区域用红框标注。
- MVP 阶段截图可暂用占位符，但 frontmatter 里的 `updated` 字段需要标记 `images: pending`。

## 十、列表与排版

- 列表项 1-2 句为佳，超过 3 句应该改写成段落。
- 不要用列表罗列段落级内容（"列表瘟疫"是教程类网站最常见的劣化）。
- 表格用于结构化对比（≥3 列且各列同构），不用于装饰。
- 引用块（`>`）用于"重要提示"或"作者私货"，每篇不超过 2 处。

## 十一、Emoji 与图标

- 章节标题、文章标题: **一律不用 emoji**。视觉一致性的底线，不可破。
- 正文: 每篇最多 5 处。可以承担情绪表达（自嘲踩坑、提示重点、轻度幽默），不只是装饰。
- **替代方案**: 结构性提示（注意 / 警告 / 详情）优先用 VitePress 的 `::: tip` `::: warning` `::: danger` `::: details` 容器，不要用 emoji 凑。
- **禁止 emoji 堆叠**: 不要 🚀🔥💯 这种连续出现，挑一个最合适的。
- **禁用过度营销 emoji**: 🎉、💎、👑、🏆 这类自嗨型 emoji 不要用。

## 十二、链接规范

- 站内链接用相对路径: `[安装 Codex CLI](../beginner/install-codex-cli.md)`。
- 站外链接首次出现要在文末或附近标注来源（不要做成裸 URL）。
- 链接文字必须有意义。"点击这里"、"详见这里"是禁用的。

## 十三、提交前自检清单（CC-Worker 必看）

提交一篇文章前对照以下清单，每条都打勾才能交付：

- [ ] frontmatter 五个字段都填了
- [ ] 标题层级不超过 H3
- [ ] 文章结构符合六段式（或概念类例外）
- [ ] 所有代码块都标了语言
- [ ] 命令行示例不带 `$` 前缀
- [ ] 至少列了 2 个常见坑
- [ ] "下一步"段落链接到 1-3 篇相关文章
- [ ] 术语用法符合 `glossary.md`
- [ ] 全文人称统一为"你"
- [ ] 没有 SEO 黑话（"小白"、"一文吃透"、"震惊"等）
- [ ] 没有 emoji 堆叠或营销型 emoji
- [ ] 网络化表达（"真香"、"翻车"等）每篇 ≤ 2 处

## 十四、代码提交前自检清单（Codex Worker 必看）

- [ ] 本地 `pnpm dev`（或 `npm run dev`）可正常启动
- [ ] 本地构建 `pnpm build` 通过
- [ ] 新增的配置项有注释或在任务卡里说明
- [ ] 没有写死的本地路径或个人 API key
- [ ] 不修改 `.agents/` 中的受保护文件
- [ ] commit 信息符合规范
- [ ] 任务卡进度日志里写了本地验证步骤
