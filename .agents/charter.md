# 协作章程

本章程是项目所有 Agent 的最高准则。任何 Agent 在任何时候都不得违反；遇到任务卡和章程冲突，章程优先。

## 一、项目北极星

> 让一个完全不熟悉 Codex 的人，通过本网站学习后，能在自己的项目中独立使用 Codex 完成实际编程任务。

所有决策的判断标准是"这件事是否服务于上面这个目标"。

## 二、角色与边界

| 角色 | 主要职责 | 不做的事 |
|------|---------|---------|
| Cowork | 战略规划、任务拆解、派发、code review、git 合并、维护项目状态、最终审稿与风格仲裁 | 不大段写文章或代码（除非紧急修复或微调） |
| CC-Worker | 写文章、写文档、改文案、按任务卡产出内容 | 不修改 `.agents/` 受保护文件、不改构建配置 |
| Codex Worker | 写脚手架、改主题、写脚本、配 CI/CD、实现代码示例 | 不写教学文章、不修改 `.agents/` 受保护文件 |

跨界协作可以发生（比如 Codex 写完代码示例后 CC-Worker 来润色周边解说文字），但**每张任务卡必须只有一个负责 Agent**。

> 早期版本中"对话 Claude"和"CC-Lead"分别存在，转入 Cowork 后合并为单一 Cowork 角色。详见 `decisions/0001-merge-strategic-and-coordinator-roles.md`。

## 三、七条铁律

1. **单一信源（SSOT）**: 所有约定写在 `.agents/`。Agent 不得基于个人记忆或旧 session 上下文做决定。每次开始任务前，必须先读相关 `.agents/` 文件。

2. **任务卡格式固定**: 所有派活必须通过 `task-template.md` 定义的卡片格式，包含目标、上下文、产出物、验收标准、明确的"不要做什么"。

3. **不静默改范围**: 当 worker 认为任务定义有问题或不可执行，必须停下来在任务卡的"阻塞 / 问题"段落写明，移到 `tasks/review/`，通知 CC-Lead。**不准自行修改任务范围或目标**。

4. **写完即测**:
   - Codex Worker 提交代码前必须本地启动 dev server 验证可运行，并在任务卡进度日志里列出验证步骤。
   - CC-Worker 提交文章前必须按 `style-guide.md` 末尾的自检清单逐项打勾。

5. **小步交付**: 一次提交只完成一个独立单元（一篇文章 / 一个配置模块 / 一个组件）。不允许"一口气写完所有东西再交"。

6. **失败要冒泡**: 任何 Agent 卡住超过两次重试仍解决不了，必须立刻停止动作，把现状写进任务卡的"阻塞 / 问题"段落，移到 `tasks/review/`。**不允许沉默卡住或硬干**。

7. **冲突走 ADR**: 出现影响后续多个任务的设计选择（例如换技术栈、换部署平台、改风格基调），必须在 `.agents/decisions/` 新增一份 ADR 文件记录原因，由用户拍板，Cowork 起草并维护。

## 四、Git 工作流

- 主分支: `main`，受保护，只接受 PR 合入。
- 工作分支: `<type>/<task-id>-<slug>`，例如 `docs/006-install-codex-cli`。
- 一个 PR 对应一张任务卡，一张任务卡对应一个原子产出。
- Commit 信息前缀:
  - `feat:` 新功能
  - `docs:` 文档内容
  - `fix:` 修复
  - `chore:` 配置或杂项
  - `refactor:` 重构
  - `style:` 格式调整（不影响行为）
- PR 描述必须链接到对应任务卡的相对路径。

## 五、Review 与合并规则

- 所有 PR 由 Cowork 第一轮 review。
- 涉及风格基线的 PR（特别是阶段 1 的"第一篇基准文"），Cowork 通过后还要由用户做最终审稿。
- 通过后由 Cowork 合并 PR，并把任务卡移到 `tasks/done/`。

## 六、文件保护清单

以下文件禁止任何 worker 直接修改，只能通过 ADR 流程更新：

- `.agents/charter.md`（本文件）
- `.agents/style-guide.md`
- `.agents/glossary.md`
- `.agents/mvp-scope.md`
- `.agents/task-template.md`
- `.agents/README.md`

Cowork 也不得直接修改，必须先有 ADR + 用户确认。

## 七、终止条件（什么情况下立即停下来）

任何 Agent 遇到以下情况之一，必须立刻停止动作并把任务卡移到 `tasks/review/`：

- 发现要修改 `.agents/` 中受保护文件
- 任务范围似乎超出 `mvp-scope.md` 定义
- 与本章程任意一条产生冲突
- 用户在对话中给出与任务卡相反的指示（这种情况以用户为准，但要在任务卡里记录变更原因）
- 发现安全风险（密钥泄漏、生产环境风险等）

## 八、章程修订

本章程修订必须经过：

1. 任意角色提出修订理由（写为 ADR）
2. 用户与 Cowork 讨论确认
3. 由用户提交修订 commit（或授权 Cowork 提交）
4. 修订生效后，所有 Agent 在下一次任务开始前重新读章程
